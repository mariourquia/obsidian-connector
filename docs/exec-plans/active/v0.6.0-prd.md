# v0.6.0 -- Live Retrieval, Safe Writes, Multi-Vault Workflows

## Problem statement

obsidian-connector v0.5.0 has breadth: 47 MCP tools, 35 CLI commands, 13 skills,
13 vault presets, and 5 distribution surfaces. But daily operation at scale
exposes three trust gaps:

1. **Write safety**: The connector increasingly writes on its own (idea routing,
   vault factory, project sync, vault guardian). Mutating commands can conflict
   with iCloud or Obsidian Sync. There is no rollback, no preview, no approval
   gate for risky writes. Roadmap item #2 (P0).

2. **Stale retrieval**: The graph index only rebuilds on explicit command. Between
   rebuilds, search and graph queries return stale results. No indicator tells
   the agent or user that the index is outdated. Roadmap items #4, #9, #22.

3. **Single-vault assumption**: The product now creates vaults (vault factory,
   presets, `/explore`), but search, graph, and workflow tools operate on one
   vault at a time with no registry, no cross-vault queries, and no per-vault
   policies. Roadmap item #5.

Secondary gaps compound these:
- Agent drafts accumulate without cleanup (roadmap #8).
- Only morning briefing has a scheduled job (roadmap #7).
- Templates are hardcoded; daily note format is not configurable (roadmap #3, #10, #15).
- Documentation counts are inconsistent across surfaces (README vs CLAUDE.md vs
  TOOLS_CONTRACT.md vs marketplace.json).

## Success criteria

| Criterion | Measurement |
|-----------|-------------|
| Zero data loss from agent writes | Pre-write snapshots exist for every mutation; rollback tested in CI |
| Index freshness < 5s after file save | Watcher detects changes and triggers incremental re-index |
| Cross-vault search works | `obsx search --vaults=all "query"` returns results from 2+ vaults |
| Draft backlog shrinks | Stale drafts auto-archived after configurable retention period |
| Doc counts match code | CI validates tool/skill/preset counts across README, CLAUDE.md, TOOLS_CONTRACT.md, marketplace.json |

## Non-goals

- Web UI or dashboard (CLI + MCP + skills remain the interfaces)
- Cloud-hosted index or sync service
- Real-time collaborative editing
- PDF export (Markdown-first; PDF deferred to v0.7.0+)
- Full policy engine UI (ship the rule layer; UI is future)
- Plugin marketplace within obsidian-connector itself (roadmap #16)

## Milestone A -- Trust and Freshness

**Theme**: Every write is safe. Every read is fresh.

### A1. Atomic write manager

**Roadmap**: #2 (P0)
**Modules affected**: `client.py`, `client_fallback.py`, `file_backend.py`, `audit.py` (new: `write_manager.py`)

All mutating operations route through a new `write_manager.py` module that
provides atomic writes and rollback.

**Requirements**:

- [ ] **A1.1** `write_manager.py`: write-then-rename for all file mutations
- [ ] **A1.2** File lock abstraction with platform-aware fallback (fcntl on POSIX, msvcrt on Windows)
- [ ] **A1.3** Pre-write snapshot: copy original to `.obsidian-connector/snapshots/{timestamp}/` before mutation
- [ ] **A1.4** Rollback command: `obsx rollback --last` restores most recent snapshot
- [ ] **A1.5** MCP tool: `obsidian_rollback` with `destructiveHint=true`
- [ ] **A1.6** Diff-before-write preview: `--preview` flag on all mutating CLI commands shows unified diff, writes nothing
- [ ] **A1.7** Approval queue: `--require-approval` flag gates root-level and template-derived bulk writes behind interactive confirmation (CLI) or explicit tool call (MCP)
- [ ] **A1.8** "Generated by agent" metadata: all agent-created files get frontmatter `generated_by: obsidian-connector` with ISO timestamp and originating tool name
- [ ] **A1.9** Protected-folder policy: `config.json` gains `protected_folders` list; writes to these folders require `--force` or approval

**Acceptance criteria**:
- `obsx create-note --preview "Test" --body "content"` prints diff, creates no file
- `obsx rollback --last` restores previous state; snapshot directory cleaned up
- Write to a protected folder without `--force` returns structured error
- `scripts/write_manager_test.py` passes with >= 20 assertions covering atomic write, lock, snapshot, rollback, preview, protected folders
- All existing mutating smoke tests still pass

**Architecture notes**:
```
client.py / file_backend.py
    |
    v
write_manager.py  (new)
    ├── atomic_write(path, content) -> write to .tmp, rename
    ├── snapshot(path) -> copy to snapshots dir
    ├── rollback(snapshot_id) -> restore from snapshot
    ├── preview(path, content) -> return unified diff string
    ├── check_protected(path, config) -> raise if protected
    └── acquire_lock(path) -> platform-aware file lock
    |
    v
audit.py  (existing, gains rollback entries)
```

### A2. File-watching index updates

**Roadmap**: #4 (P1), #22 (backlog)
**Modules affected**: `index_store.py`, `graph.py`, `config.py` (new: `watcher.py`)

**Requirements**:

- [ ] **A2.1** `watcher.py`: filesystem watcher using `watchdog` (optional dependency, graceful degradation if not installed)
- [ ] **A2.2** Watch vault directory for `.md` file creates, modifies, deletes
- [ ] **A2.3** On change: incremental re-index of affected files only (existing `index_store.py` already does mtime-based incremental -- wire it to watcher events)
- [ ] **A2.4** Stale-index indicator: every MCP/CLI response includes `index_age_seconds` in the JSON envelope when index is > 60s old
- [ ] **A2.5** CLI command: `obsx watch` starts watcher in foreground; `obsx watch --daemon` backgrounds it
- [ ] **A2.6** MCP tool: `obsidian_index_status` returns last rebuild time, file count, staleness flag
- [ ] **A2.7** Watcher respects `.gitignore`-style exclude patterns from `config.json`

**Acceptance criteria**:
- Create a file in the vault while watcher runs; `obsx search` finds it within 5 seconds
- Delete a file; search no longer returns it within 5 seconds
- `obsx search --json "query"` includes `index_age_seconds` when > 60s stale
- `scripts/watcher_test.py` passes (mocked filesystem events, no real watcher needed for CI)

**Architecture notes**:
```
watcher.py (new)
    ├── start_watching(vault_path, callback)
    ├── stop_watching()
    └── on_change(event) -> index_store.update_file(path)

index_store.py (existing)
    └── update_file(path)  -- already does mtime check, just needs public entry point

envelope.py (existing)
    └── success() gains optional index_age_seconds field
```

**Dependency**: `watchdog` added to `[project.optional-dependencies]` as `live = ["watchdog>=4.0,<5.0"]`.

## Milestone B -- Smarter Retrieval and Vault Scale

**Theme**: Find anything across any vault. Know why it was returned.

### B1. Hybrid retrieval

**Roadmap**: #9 (P1)
**Modules affected**: `search.py`, `graph.py`, `index_store.py` (new: `retrieval.py`, `embeddings.py`)

**Requirements**:

- [ ] **B1.1** `retrieval.py`: unified retrieval interface that combines multiple signals
- [ ] **B1.2** Signal sources:
  - Lexical: existing keyword search (BM25-style scoring from `search.py`)
  - Semantic: local embedding similarity (new `embeddings.py`)
  - Graph: backlink count and neighborhood centrality boost from `graph.py`
  - Recency: mtime-based decay factor
- [ ] **B1.3** `embeddings.py`: local embedding generation using `sentence-transformers` (optional dependency). Falls back to lexical-only when not installed.
- [ ] **B1.4** Embedding index stored in SQLite alongside graph index (new table in `index_store.py`)
- [ ] **B1.5** Retrieval explanation: each result includes `match_reasons` list (e.g., `["keyword: 'portfolio'", "backlink_boost: 3 inbound", "recency: modified 2h ago"]`)
- [ ] **B1.6** Retrieval profiles via `--profile` flag:
  - `journal`: boost recency, prefer daily notes
  - `project`: boost backlink density, prefer project folders
  - `research`: boost semantic similarity, prefer long-form notes
  - `review`: balanced, include stale notes
- [ ] **B1.7** MCP: `obsidian_search` gains optional `profile` and `explain` parameters
- [ ] **B1.8** CLI: `obsx search --profile research --explain "query"`

**Acceptance criteria**:
- `obsx search --explain --json "query"` returns results with `match_reasons` array
- `obsx search --profile journal "query"` returns different ranking than `--profile research`
- Without `sentence-transformers` installed, search still works (lexical + graph + recency only)
- `scripts/retrieval_test.py` passes with >= 25 assertions

**Architecture notes**:
```
retrieval.py (new)
    ├── search(query, profile, explain) -> list[SearchResult]
    ├── _lexical_score(query, note) -> float
    ├── _semantic_score(query, note) -> float  (optional)
    ├── _graph_score(note) -> float
    ├── _recency_score(note) -> float
    └── _merge_scores(signals, profile_weights) -> ranked results

embeddings.py (new)
    ├── embed(text) -> list[float]
    ├── build_index(vault_path) -> None  (stores in SQLite)
    └── similar(query_embedding, top_k) -> list[(note_path, score)]

index_store.py (existing)
    └── new table: embeddings(note_path TEXT, embedding BLOB, updated_at REAL)
```

**Dependency**: `sentence-transformers` added to `[project.optional-dependencies]` as `semantic = ["sentence-transformers>=3.0,<4.0"]`.

- [ ] **B2.8** Per-vault policy overrides in registry: `protected_folders`, `draft_max_age_days`, `watcher_enabled`
- [ ] **B2.9** Vault picker: `obsx vaults pick` interactive selector; MCP equivalent returns vault list for agent to choose

**Acceptance criteria**:
- `obsx vaults add work ~/vaults/work && obsx vaults add personal ~/vaults/personal` registers both
- `obsx search --vaults=all --json "query"` returns results from multiple vaults, each tagged with vault name

- `scripts/vault_registry_test.py` passes with >= 20 assertions

## Milestone C -- Daily Operating System

**Theme**: The vault runs itself between sessions.

### C1. Scheduler expansion

**Roadmap**: #7 (P1)
**Modules affected**: `scheduling/`, `platform.py`, `workflows.py` (new: `scheduler.py`)

**Requirements**:

- [ ] **C1.1** `scheduler.py`: first-class scheduler config in `config.json` under `schedules` key
- [ ] **C1.2** Built-in schedule types: `morning` (existing), `evening`, `weekly`
- [ ] **C1.3** Per-vault schedules: each vault in the registry can enable/disable specific schedules
- [ ] **C1.4** Workflow chaining: schedules can define a sequence of tools to run (e.g., morning: `[check_in, sync_projects, today]`)
- [ ] **C1.5** Event triggers: `after_sync`, `after_note_create`, `after_session_end` hooks in config that run a tool chain
- [ ] **C1.6** Dry-run preview: `obsx schedule preview morning` shows what would run without executing
- [ ] **C1.7** Health status: `obsx schedule status` shows last-run, next-run, missed-run for each schedule
- [ ] **C1.8** Missed-run recovery: if a schedule was missed (machine off), run on next opportunity with `--catchup` flag
- [ ] **C1.9** Active hours window: `config.json` gains `active_hours` (default: `"07:00-22:00"`) -- schedules outside this window are deferred

**Acceptance criteria**:
- `obsx schedule preview morning --json` returns ordered tool list without side effects
- `obsx schedule status --json` shows schedule health for all configured schedules
- `scripts/scheduler_test.py` passes with >= 15 assertions (mocked time, no real scheduling)

### C2. Template system

**Roadmap**: #3 (P0), #10 (P1), #15 (P1)
**Modules affected**: `config.py`, `vault_presets.py`, `client.py`, `file_backend.py` (new: `template_engine.py`)

**Requirements**:

- [ ] **C2.1** `template_engine.py`: load templates from vault's `_templates/` folder with variable substitution (`{{date}}`, `{{title}}`, `{{vault_name}}`, `{{project}}`, custom vars)
- [ ] **C2.2** Built-in workflow templates: meeting note, research note, decision log, project idea, daily note
- [ ] **C2.3** Configurable daily note path: `config.json` gains `daily_note_path` (default: `"daily/{{date}}.md"`) and `daily_note_format` (default: `"YYYY-MM-DD"`)
- [ ] **C2.4** Configurable sentinel headings: `config.json` gains `sentinels` map (e.g., `{"morning_ritual": "## Morning", "evening_ritual": "## Evening"}`)
- [ ] **C2.5** Template inheritance: templates can extend a base template with `extends: base-note` in frontmatter, overriding only specific sections
- [ ] **C2.6** Template versioning: templates include `version` in frontmatter; `obsx templates check` warns when vault templates are outdated vs built-in
- [ ] **C2.7** CLI: `obsx templates list`, `obsx templates init` (seed `_templates/` from built-ins), `obsx templates check`
- [ ] **C2.8** MCP: `obsidian_list_templates`, `obsidian_create_from_template`
- [ ] **C2.9** Migration helper: `obsx templates migrate` detects hardcoded daily note paths and sentinel headings in existing vault files, reports what would change

**Acceptance criteria**:
- `obsx create-note --template meeting --json` creates note from template with variables substituted
- Changing `daily_note_format` in config.json changes the filename format for `obsx log-daily`
- Changing a sentinel in config.json changes what `workflows.py` ritual detection looks for
- `scripts/template_test.py` passes with >= 20 assertions

## Risks and mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| `watchdog` compatibility across macOS/Linux/Windows | Watcher fails on some platforms | Optional dependency; falls back to manual `rebuild-index` |
| `sentence-transformers` is large (~500MB with model) | Install friction for non-ML users | Optional dependency; lexical search works without it |
| File lock contention with Obsidian Sync | Deadlock or data corruption | Lock timeout (5s default); fallback to copy-on-write without lock |
| Multi-vault registry path changes | Broken references after vault move | `obsx vaults doctor` detects unreachable vaults and prompts repair |
| Scope creep across 4 milestones | Release delayed | Milestones are independently shippable; can release A+B as v0.6.0 and C+D as v0.6.1 |

## Cut list (defer to v0.7.0+ if scope is tight)

**Safe to defer**:
- PDF export (C4 stays Markdown-only)
- Advanced graph-quality tooling (orphan detection, duplicate concept detection)
- Preset marketplace / org-level preset packs
- Full policy engine (ship `protected_folders` only; broader rule engine is future)
- Knowledge graph quality tools (backlink opportunity suggestions, missing-MOC suggestions)
- Event-triggered automation (ship schedule expansion; event hooks are v0.7.0)

**Do not defer**:
- Write safety (A1) -- trust foundation for everything else
- File-watching index (A2) -- retrieval freshness is table stakes
- Draft lifecycle (A3) -- agent hygiene before the vault degrades
- Multi-vault foundation (B2) -- vault factory already creates multiple vaults
- Docs cleanup (D1) -- counts are already drifting
