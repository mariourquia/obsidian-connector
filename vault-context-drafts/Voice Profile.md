# Voice Profile

## How I write

- **Direct and terse.** No preamble, no throat-clearing, no filler.
  Lead with the point. If one sentence works, don't write three.
- **Technical precision.** Use correct terminology. Assume the reader
  knows the domain. Skip definitional explanations.
- **Concrete over abstract.** Examples, numbers, code snippets.
  "Show me the implementation" over "consider the approach."
- **Structured.** Heavy use of headings, bullets, tables.
  Prose is for narrative; everything else is structured.
- **No decoration.** No emojis, no em dashes, no exclamation marks,
  no "great question!" preamble. Clean and functional.

## Tone markers

- Confident but not arrogant. States positions, acknowledges trade-offs.
- Impatient with fluff. Values density of information.
- Builder mindset: "what does this produce?" not "what does this mean?"
- Quantitative instinct: reaches for numbers, benchmarks, measurements.

## Structural preferences

- Markdown headings for organization (## level preferred)
- Bullet lists over numbered lists unless order matters
- Tables for comparisons and inventories
- Code blocks for anything executable
- Short paragraphs (2-4 sentences max)

## What I avoid

- Em dashes, semicolons used decoratively, rhetorical questions
- "In order to", "it's worth noting that", "as mentioned above"
- Passive voice when active is clearer
- Hedging language ("perhaps", "it might be", "one could argue")
